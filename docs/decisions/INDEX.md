# Decisions - Index

Un **ADR** (Architecture Decision Record) documente une decision d'architecture significative : pourquoi elle a ete prise, dans quel contexte, et quelles en sont les consequences. Format Michael Nygard. On en ecrit un des qu'une decision est non triviale, difficile a renverser, ou susceptible de revenir en question.

Regle : avant toute decision d'architecture non triviale, relire cet index.

Les ADR sont crees via le skill `adr-new` (a venir).
